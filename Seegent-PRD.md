# Seegent — 带 AI 的本地可视化工作台

> **版本**: v0.2 | **更新**: 2026-06-30

## 1. 定位

**你的文件 + 你的 AI Key + 你的规则 = 你的工作台。没人能看到你的数据。**

Seegent 是一个浏览器端运行的可视化工作台。你管理本地文件，你配置自己的 AI 模型和 API Key，你定义 AI 的行为规则。所有数据留在你的机器上——文件内容不进云端存储，对话历史不进第三方服务器，LLM 调用直连你配置的 provider。

和 Claude Code / Codex / Cursor 的本质区别：它们是你**托付数据**给平台；Seegent 是你**掌握数据**，AI 来你的机器上工作。

## 2. 目标用户

- 对隐私敏感的知识工作者（写作者、研究员、独立开发者）
- 需要 AI 辅助管理大量本地文件，但不愿意把文件上传到云端 AI 服务的人
- 已经有一个或多个本地文件夹的工作流，想在此基础上加 AI 能力，而不是迁移到新平台

## 3. 核心价值

### 3.1 隐私架构

```
你的浏览器（IndexedDB）        你的机器（本地文件系统）
      │                              │
      │ File System Access API       │ Python 后端
      │ （文件不经过服务器）           │ （工具执行在工作区）
      │                              │
      └──────────┬───────────────────┘
                 │
         你配置的 LLM API
         （直连，不经过 Seegent 中转存储）
```

- **文件内容**：通过浏览器 File System Access API 直读，不进云端
- **对话历史**：存本地 JSON 文件，不经过第三方
- **LLM 调用**：直连你配置的 provider（OpenAI / DeepSeek / 302.AI / 任意兼容 API）
- **API Key**：你填你管，不存明文在代码仓库
- **向量索引**：SQLite 本地存储，查询不调云端 API

### 3.2 Agent 能在你的文件上做什么

聊天框里用自然语言，Agent 可以：

| 能力 | 工具 |
|------|------|
| 浏览目录结构 | `list_dir` |
| 读取文件（支持分段读长文件） | `read_file` |
| 写入/创建文件 | `write_file` |
| 全文搜索（行号 + 上下文） | `search_files` |
| 语义搜索（"我去年写的关于定价的笔记"） | `semantic_search` |
| 文件统计 | `file_stats` |
| 最近修改文件 | `recent_files` |
| 网页搜索 | `web_search` |
| 网页抓取 | `web_fetch` |
| 执行 Shell 命令 | `run_shell`（待实现） |

加上 MCP 协议扩展（飞书、企业微信、Notion 等），工具可无限扩展。

### 3.3 可视化文件管理

终端系 AI 工具（Claude Code、Codex）没有的东西：

- 树形文件夹浏览 + 拖拽排序
- Markdown 编辑 / 实时预览
- HTML 文件实时预览（iframe + 文件监听）
- 标签页式多文件编辑
- 刷新/重启后文件夹状态完整恢复（IndexedDB 持久化）

## 4. 非目标（明确不做）

| 不做的事 | 理由 |
|---------|------|
| 多 Agent 协作/编排（invoke_agent） | 单 Agent + 好工具已经足够强，多 Agent 增加复杂度但没有隐私增益 |
| 社交媒体数据监测 dashboard | 属于另一个产品方向，放在一起模糊定位 |
| Agent 市场/社区分享 | 本地优先工具不需要 App Store |
| 远程 Agent 调用 | 违背"数据不出本地"原则 |
| 自动路由/意图识别 | 用户自己决定让 AI 做什么，不过度"智能化" |

## 5. 功能范围

### 已实现

- 文件树管理（浏览/编辑/预览/新建/重命名/删除/拖拽排序）
- Markdown 编辑预览 + HTML 实时预览
- 多模型支持（任意 OpenAI 兼容 provider）
- Agent 循环（think → act → observe → think），SSE 流式推送
- 9 个 Agent 工具（见 3.2）
- 语义搜索（SQLite 向量索引 + 可配置 embedding）
- 多模态消息（粘贴/拖拽图片 → vision API）
- 角色系统（预置 12 个角色，可自定义提示词）
- MCP CLI 集成（飞书、企业微信、Notion 等）
- 文件夹路径绑定（浏览器 handle ↔ 后端工作区路径）

### 待实现（按优先级）

| 优先级 | 功能 | 价值 |
|--------|------|------|
| P0 | Shell 命令执行（`run_shell`） | 解锁 git/npm/pip/脚本——一个工具等于无数工具 |
| P1 | 精确编辑工具（字符串替换，替代全量覆写） | 让 Agent 改代码更精准，减少误写 |
| P1 | Agent 并行工具调用 | 一轮对话同时查文件 + 搜索，速度质的提升 |
| P2 | 批量文件操作 | 移动/删除/重命名多个文件 |
| P2 | 图片处理（Pillow） | 生成封面图、调整尺寸 |
| P3 | 工作区多路径支持 | 跨项目工作 |
