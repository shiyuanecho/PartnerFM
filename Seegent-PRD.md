# Seegent — 项目管理指挥部

> **版本**: v1.0 | **更新**: 2026-07-02
>
> 这是 Seegent 的**主 PRD**。功能特性层面的详细文档请参考子文档。
>
> **核心定位变更**（v0.3 → v1.0）：从模糊的"AI 工作台"收敛为**项目管理指挥部**。Seegent 负责管理项目、追踪进度、组织文件；写代码/做图/批量自动化等具体执行由外部智能体（WorkBuddy、Cursor）完成。

---

## 1. 定位

**Seegent = 项目管理指挥部。外部 AI 工具 = 干活的兵。**

```
你在 Seegent 里：                          外部智能体帮你：
┌─────────────────────────┐                ┌──────────────────┐
│ 📊 看数据 → 做决策       │                │ 💻 写代码          │
│ 📋 理待办 → 排优先级      │  ── 分配任务 ─→ │ 🎨 做图            │
│ 📁 管文件 → 看产出物      │                │ ⚡ 批量自动化      │
│ 🤖 AI 助 → 帮分析/总结   │  ←── 产出落回 ── │ 📝 写文档          │
└─────────────────────────┘                └──────────────────┘
```

和 Claude Code / Cursor 的关系不是竞争，是**配合**；它们干活，Seegent 管活。

### 一句话

> Seegent 是你每天第一件事打开的地方——看数据、管待办、分配任务、验收产出。实际执行交给 WorkBuddy 等外部智能体。

---

## 2. 目标用户

1. **独立开发者**（如我，拾元老师）——同时跑 2-3 个项目，需要一眼看清每个项目的状态
2. 未来可能：其他同时管理多个小项目的独立创作者

---

## 3. 核心价值

### 3.1 项目指挥部

| 我在 Seegent 做什么 | 用什么功能 |
|-----|-----------|
| 每天早上看各项目状态 | 🏠 首页看板 |
| 看业务数据趋势 | 📊 看板指标卡片 |
| 梳理今天要做什么 | 📋 待办汇总 |
| 查看最近产出了什么 | 📂 最近活动 / 文件管理 |
| 让 AI 帮我分析数据/总结进展 | 💬 AI 对话 |
| 连接外部数据源（飞书多维表格） | 🔌 数据源 / MCP |
| 把任务分配给外部 AI 去执行 | 切到 WorkBuddy/Cursor，产出落回文件夹 |

### 3.2 隐私架构

```
你的浏览器（IndexedDB）        你的机器（本地文件系统）
      │                              │
      │ File System Access API       │ Python 后端
      │ （文件不经过服务器）           │ （工具执行在工作区）
      │                              │
      └──────────┬───────────────────┘
                 │
         你配置的 LLM API
         （直连，不经过 Seegent 中转存储）
```

- 文件、对话、索引全部在本地
- API Key 你填你管，不进代码仓库

### 3.3 内置 AI 的角色

> Seegent 的内置 AI 不写代码、不做图、不跑自动化。它的职责是**帮你管项目**。

| 能力 | 示例 |
|------|------|
| 📊 分析数据 | "帮我分析这周识字 App 的数据趋势" |
| 📝 总结进展 | "总结一下 Seegent 这个月的开发进展" |
| 🔍 找文件/信息 | "我去年写的关于定价的笔记在哪" |
| 📋 整理待办 | "从 TODO.md 里提取所有未完成的任务" |
| 📈 生成周报 | "根据最近两周的产出，生成工作周报" |

## 4. 非目标（明确不做）

| 不做的事 | 理由 |
|---------|------|
| 替代外部智能体执行具体任务（写代码/做图/自动化） | 那是 WorkBuddy/Cursor 的活，Seegent 是管理者不是执行者 |
| Shell/终端能力 | 项目管理不需要跑脚本 |
| 多 Agent 协作编排 | 一个管理型 Agent 足够 |
| 社交媒体数据监测 | 独立产品方向，不做 |
| Agent 市场/社区分享 | 工具不需要 App Store |
| 自动路由/意图识别 | 用户自己决定让 AI 做什么 |

## 5. 功能范围

### 已实现

| 功能 | 说明 |
|------|------|
| 📊 项目看板 V1 | 绑定文件夹 → 项目卡片，飞书多维表格数据源 |
| 📁 文件管理 | 树形浏览、多标签编辑、Markdown/HTML 预览、拖拽排序 |
| 💬 AI 对话 | 多会话、流式输出、多模型切换、多模态图片 |
| 🔍 全文搜索 | 关键字搜索 + 行号上下文 |
| 🧠 语义检索 | SQLite 向量索引，"我去年写的定价笔记" |
| 🔌 数据源接入 | 飞书多维表格自动同步指标 |
| 🎭 角色系统 | 预置角色 prompt |
| 🔌 MCP 集成 | 飞书、企业微信、Notion 等外部工具 |

### 待实现（按优先级）

| 优先级 | 功能 | 价值 |
|--------|------|------|
| **P0** | 🏠 **看板首页升级** | 打开 Seegent 直接看到所有项目状态 + 待办 + 最近活动，不再先看到文件夹。实现方案见 `看板升级-首页改造-实现方案.md` |
| **P1** | 📋 **待办管理增强** | 从文件自动扫描 TODO，在看板中聚合展示，支持勾选完成 |
| **P1** | 📈 **AI 项目分析** | 内置 AI 分析数据趋势、生成周报、对比项目进展 |
| **P2** | 🎭 **角色精简** | 从 12 个角色压缩到 3-5 个（项目管理、数据分析、内容审阅），删掉代码/作图相关角色 |
| **P2** | ⚙️ **设置整合** | 模型/角色/MCP 入口收进统一设置页 |

## 6. 系统架构

```
┌────────────────────────────────────────────┐
│              Seegent 浏览器端              │
│  ┌──────────┐  ┌──────────┐  ┌───────────┐ │
│  │ 🏠 看板   │  │ 📁 文件   │  │ 💬 AI 对话 │ │
│  │ (首页入口) │  │ (树形+标签)│  │ (项目管理) │ │
│  └──────────┘  └──────────┘  └───────────┘ │
│  ┌──────────┐  ┌──────────────┐           │
│  │ 🔧 设置   │  │ 🔌 MCP 管理  │           │
│  └──────────┘  └──────────────┘           │
│        │            │ SSE           │ REST  │
└────────┼────────────┼───────────────┼───────┘
         │            │               │
   File System API     │               │
         │            │               │
┌────────┴────────────▼───────────────▼───────┐
│           Seegent Server (Python)          │
│  ┌──────────┐  ┌──────────┐  ┌────────────┐ │
│  │ Agent    │  │ LLM 代理  │  │ MCP 客户端  │ │
│  │ (管理工具) │  │ (多模型)  │  │ (飞书等)   │ │
│  └──────────┘  └──────────┘  └────────────┘ │
│  ┌──────────┐  ┌──────────┐                 │
│  │ 语义检索  │  │ 文件工具  │                 │
│  │ (SQLite) │  │ (os.walk)│                 │
│  └──────────┘  └──────────┘                 │
└──────────────────────────────────────────────┘

外部智能体（WorkBuddy / Cursor / Claude Code）
     ↕ 读写同一个文件夹
你机器上的项目文件夹
```
