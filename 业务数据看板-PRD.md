# Seegent 业务数据看板 — 产品需求文档

> **版本**: v0.2 | **状态**: 已确认方向 | **更新**: 2026-06-28
>
> 从业务结果数据驱动内容产出与行动决策。

---

## 1. 问题陈述

**用户场景**：拾元运营着多个产品（识字 App、Learn Chinese 等），各产品的业务数据散落在不同平台后台（小红书导出 CSV、飞书多维表格等）。每次想看数据要逐个登录平台，数据之间割裂，也无法快速从数据中获取洞察并驱动下一步行动。

**当前缺陷**：
- 没有统一的业务数据视图，各平台数据各自为政
- 看到数据异常/机会后，需要手动整理数据 → 打开聊天 → 把数据喂给 Agent → 等待分析，流程长且容易遗漏
- Agent 分析数据时缺乏结构化上下文，每次都要用户手动描述"现在 DAU 是这样、收入是那样"
- 无法追踪"这个数据洞察产出了什么行动"，数据 → 分析 → 决策 → 执行之间是断开的
- 不同项目的指标混在一起没有隔离，容易混淆

**不解决的成本**：
- 业务决策靠零散记忆，无系统化复盘
- 数据异常发现滞后，错过最佳响应窗口
- Agent 没有被充分用于数据分析——明明有强大的分析能力，但喂数据的环节太痛苦

## 2. 目标

| # | 目标 | 可衡量标准 |
|---|------|-----------|
| 1 | 每个项目有自己独立的看板，集中展示该项目的核心业务指标 | 打开项目看板，5 秒内获取该项目业务健康度概况 |
| 2 | 一键触发 Agent 对任意指标做深度分析 | 点击「分析」按钮后，Agent 在 30 秒内产出结构化分析报告 |
| 3 | Agent 自主选择分析维度与方法 | 用户只需触发，Agent 根据数据上下文自动决定分析角度（趋势/对比/归因/预测） |
| 4 | 支持 CSV 文件上传 + 飞书多维表格同步两种数据接入方式 | 上传 CSV → 自动解析列 → 生成卡片；连接飞书表格 → 自动同步 |
| 5 | 分析报告存入项目文件夹，可追溯 | 每次分析报告保存在 `{项目}/产出/数据/` 目录下 |

---

## 3. 非目标

| # | 非目标 | 理由 |
|---|--------|------|
| 1 | 不做实时数据推送/WebSocket | 数据以 CSV 上传为主，飞书同步采用手动拉取，v1 不需要实时流 |
| 2 | 不做 BI 级别的拖拽式数据探索 | Seegent 是 AI 工作台，不是 BI 工具。深度探索交给 Agent |
| 3 | 不做自动告警/阈值通知 | v1 用户主动查看 + 触发分析，Phase 2 再加自动化规则 |
| 4 | 不替代各平台原生后台 | 看板是聚合视图，精细化操作仍回原平台 |
| 5 | 不做数据写入/回写 | 只读不写，不会反向修改外部平台数据（包括飞书表格） |
| 6 | 不做多用户协作/权限 | Seegent 是单用户本地应用 |
| 7 | **分析和执行不直接挂钩** | 分析结果先看，满意后用户手动决定是否执行，中间不自动跳转 |
| 8 | 不做跨项目数据聚合 | 每个项目独立看板，不做「识字 + Learn Chinese」合并视图 |

## 4. 核心概念模型

```
┌──────────────────────────────────────────────────────────────┐
│                    Seegent 项目结构                           │
│                                                               │
│  📁 识字App（项目文件夹）         📁 Learn Chinese（项目文件夹）  │
│  ├── 📊 看板（独立）              ├── 📊 看板（独立）            │
│  ├── 产出/                       ├── 产出/                     │
│  │   └── 数据/    ← 分析报告      │   └── 数据/    ← 分析报告    │
│  └── ...                         └── ...                      │
└──────────────────────────────────────────────────────────────┘

> **入口**：看板通过顶栏全局导航「📊 看板」模块进入；模块顶部「项目选择器」切换项目，加载该项目的独立看板配置。入口全局化，数据仍按项目隔离。

每个项目的看板独立运作：

  ┌──────────────┐    ┌──────────────┐    ┌───────────────┐
  │  CSV 上传     │    │  飞书多维表格  │    │  手动输入      │
  │ (小红书导出)  │    │  (自动同步)   │    │  (零散指标)    │
  └──────┬───────┘    └──────┬───────┘    └───────┬───────┘
         │                  │                    │
         ▼                  ▼                    ▼
  ┌──────────────────────────────────────────────────┐
  │              数据解析 & 缓存层                     │
  │   .seegent-dashboard.json（项目级配置文件）       │
  └──────────────────────┬───────────────────────────┘
                         │
             ┌───────────┴───────────┐
             ▼                       ▼
  ┌──────────────────┐    ┌──────────────────┐
  │   指标卡片网格     │    │   智能分析触发     │
  │   粉丝 📈         │    │   → 选择 Agent    │
  │   曝光 📊         │    │   → 注入数据上下文  │
  │   互动 💬         │    │   → Agent 自主分析  │
  │                   │    │   → 报告存入项目目录 │
  └──────────────────┘    └──────────────────┘
```

**关键概念**：

| 概念 | 定义 | 类比 |
|------|------|------|
| **看板（Dashboard）** | 绑定到一个项目文件夹的指标集合，每个项目一个看板 | 一个 Grafana Dashboard |
| **数据源（DataSource）** | 指标的来源：CSV 文件（上传解析）/ 飞书多维表格（同步）/ 手动输入 | 数据库连接 |
| **指标卡片（MetricCard）** | 看板上的一个可视化单元：指标名 + 当前值 + 趋势 + 操作按钮 | Grafana Panel |
| **数据快照（Snapshot）** | 某个时间点导入的原始数据，保留历史用于趋势对比 | 数据库行 |
| **分析任务（AnalysisTask）** | 用户触发的一次 Agent 分析：绑定指标 + Agent → 产出分析报告到项目文件夹 | 工作流中的一步 |

### 5.2 数据源类型详解

| 类型 | 标识 | 数据来源 | 适用场景 | v1 支持 |
|------|------|---------|---------|---------|
| CSV 上传 | `csv_upload` | 用户从平台导出 CSV → 上传到 Seegent | 小红书后台、抖音后台等支持 CSV 导出的平台 | ✅ P0 |
| 飞书多维表格 | `feishu_bitable` | 连接飞书多维表格，手动触发同步 | 已经在飞书表格里维护的运营数据 | ✅ P0 |
| 手动输入 | `manual` | 前端输入端填数据 | 零散指标，无系统来源 | P1 |
| 自定义 API | `custom_api` | HTTP GET 请求拉 JSON | 自有后端分析接口 | P1 |
| MCP 数据源 | `mcp` | 通过 MCP 协议连接外部数据源 | 通过现有 MCP 体系扩展 | P2 |

### 5.4 分析记录（存入项目文件夹 `产出/数据/` 目录）

分析记录本身是轻量索引，存储在看板配置中；分析报告文件存在项目目录下。

```json
{

    {
      "id": "analysis-001",
      "cardId": "card-001",
      "agentId": "data-analyst",
      "agentName": "数据分析师",
      "triggeredAt": "2026-06-28T10:30:00+08:00",
      "dataContext": {
        "current": 12580,
        "previous": 12400,
        "trend": [12000, 12100, 12250, 12400, 12580]
      },
      "reportPath": "产出/数据/小红书粉丝分析_20260628.md",
```

> **报告存储规则**：`{项目文件夹}/产出/数据/{指标名}_分析_{日期}.md`
> 示例：`/Users/shiyuanchang/识字App/产出/数据/小红书粉丝_分析_20260628.md`

## 6. 用户故事

### P0 — 必须实现

| ID | 用户故事 | 验收标准 |
|----|---------|---------|
| US-01 | **作为运营者**，我希望在项目文件夹里打开看板，看到该项目绑定的指标卡片（从 CSV 或飞书表格解析出的指标），以便快速了解该项目的业务概览 | [ ] 看板绑定到项目文件夹<br>[ ] 指标卡片网格展示<br>[ ] 每张卡片显示：指标名、当前值、环比变化、迷你趋势图<br>[ ] 变化方向用颜色区分（绿涨红跌） |

## 7. 功能需求

### 7.1 Must-Have（P0）

| 编号 | 模块 | 需求描述 |
|------|------|---------|
| R-01 | 后端 | **看板配置 API**：`GET/POST /api/dashboard/config?project=路径`，读写 `.seegent-dashboard.json`（按项目路径隔离） |
| R-02 | 后端 | **CSV 上传解析 API**：`POST /api/dashboard/upload-csv`，接收 CSV 文件 → 返回解析后的表头 + 前 10 行预览数据 |
| R-03 | 后端 | **CSV 数据确认导入 API**：`POST /api/dashboard/import-csv`，根据用户指定的列映射，将 CSV 行解析为结构化快照并存储 |
| R-04 | 后端 | **飞书多维表格同步 API**：`POST /api/dashboard/sync-feishu`，使用飞书 Open API 拉取指定表格数据，按列映射解析为快照 |
| R-05 | 后端 | **数据上下文注入**：当用户从看板触发分析时，自动将指标快照数据注入 Agent 的系统消息或第一条用户消息 |
| R-06 | 后端 | **分析报告写入**：Agent 产出分析报告后，自动保存为 `{项目路径}/产出/数据/{指标名}_分析_{日期}.md` |
| R-07 | 前端 | **导航入口**：在顶栏全局导航新增「📊 看板」模块（`data-module="board"` 按钮，已就绪），模块顶部提供「项目选择器」，按项目隔离展示对应看板。入口全局化方便随时查看，数据仍按项目隔离 |
| R-08 | 前端 | **看板视图**：响应式卡片网格布局（桌面 2-4 列，移动端 1 列），每张卡片包含：标题、当前值（大字）、环比变化（带箭头颜色）、迷你趋势图（SVG sparkline） |
| R-09 | 前端 | **CSV 上传流程**：拖拽/点击上传 CSV → 预览表格（表头 + 前 3 行）→ 用户配置列映射（日期列 + 指标列）→ 确认导入 → 自动生成卡片 |
| R-10 | 前端 | **飞书表格连接流程**：输入 App Token + Table ID → 测试连接 → 配置列映射 → 手动点击同步 → 自动生成卡片 |
| R-11 | 前端 | **卡片操作**：每张卡片右上角有「📊 分析」按钮。CSV 卡片有「🔄 重新上传」按钮。飞书卡片有「🔄 同步」按钮 |
| R-12 | 前端 | **分析触发流程**：点击分析 → 弹出 Agent 选择器（默认使用卡片配置的 analysisAgentId）→ 确认后自动切换到聊天模块 → 打开新对话或复用已有对话 → 自动发送首条消息（含数据上下文）→ Agent 开始分析 |
| R-13 | 前端 | **添加手动卡片**：「＋ 添加卡片」按钮 → 输入指标名称、单位 → 生成一张手动输入卡片（看板上直接填数值） |
| R-14 | 前端 | **看板绑定项目**：看板与 Seegent 项目文件夹一对一绑定，切换项目文件夹时自动切换对应看板 |

### 7.2 Nice-to-Have（P1）

| 编号 | 需求描述 |
|------|---------|
| R-13 | 卡片拖拽排序 + 尺寸调整 |

```python
import csv
import io

def parse_csv_preview(csv_bytes: bytes, encoding: str = 'utf-8') -> dict:
    """解析 CSV，返回表头 + 前 10 行预览"""
    text = csv_bytes.decode(encoding)
    reader = csv.DictReader(io.StringIO(text))
    headers = reader.fieldnames or []
    rows = []
    for i, row in enumerate(reader):
        if i >= 10:
            break
        rows.append(row)
    return {"headers": headers, "preview_rows": rows, "total_rows": "..."}

def import_csv_data(csv_bytes: bytes, column_mapping: dict, encoding: str = 'utf-8') -> list[dict]:
    """根据列映射将 CSV 解析为快照行列表：
    column_mapping = {
        "date": "日期",
        "metrics": {
            "followers": {"column": "粉丝总数", "unit": "人"},
            "impressions": {"column": "笔记曝光量", "unit": "次"}
        }
    }
    返回: [{"date":"2026-06-24", "followers": 12000, "impressions": 95000}, ...]
    """
    text = csv_bytes.decode(encoding)
    reader = csv.DictReader(io.StringIO(text))
    rows = []
    date_col = column_mapping["date"]
    metric_map = column_mapping["metrics"]
    for row in reader:
        snap = {}
        snap["date"] = row.get(date_col, "")
        for key, cfg in metric_map.items():
            raw_val = row.get(cfg["column"], "0")
            try:
                snap[key] = float(str(raw_val).replace(",", "").replace("%", ""))
            except ValueError:
                snap[key] = 0
        rows.append(snap)
    return sorted(rows, key=lambda r: r["date"])
```

### 11.2 卡片的环比计算规则

```
环比变化率 = (current - previous) / previous × 100%
颜色规则：
- ↑ 增长 ∧ 指标是正向指标（DAU/收入/粉丝） → 绿色
- ↓ 下降 ∧ 指标是正向指标 → 红色
- 对于负向指标（如流失率），颜色反转
```

## 14. 风险与缓解

| 风险 | 概率 | 影响 | 缓解 |
|------|------|------|------|
| CSV 格式不一致（小红书后台导出格式可能变化） | 中 | 中 | CSV 上传后先预览表头，用户手动映射列，灵活适应格式变化 |
| 飞书多维表格 API 权限复杂 | 中 | 中 | v1 手动触发同步，用户自己确保 token 有效；失败时明确报错不静默 |
| Agent 分析质量不稳定 | 中 | 中 | 提供分析提示词模板，用户可编辑调优；分析结果不满意可重新触发 |
| 数据上下文过大导致 Agent 无法处理 | 低 | 低 | 只注入最近 5-7 个时间点 + 当前/上期对比，不喂全量数据 |
| 飞书 App Token / API Key 泄露 | 低 | 高 | 看板配置文件加入 `.gitignore`，敏感字段脱敏显示 |

## 15. 已确认决策

| # | 问题 | 决策 |
|---|------|------|
| 1 | 小红书数据怎么接入？ | **CSV 导出 → 上传解析**。不从后台直接拉 API（反爬风险），用户从小红书后台导出 CSV，拖入看板 |
| 2 | 不同项目的数据合在一起还是分开？ | **按项目隔离**。每个项目文件夹绑定一个独立看板，识字 App 和 Learn Chinese 互不干扰 |
| 3 | Agent 分析报告存哪里？ | **项目文件夹内**：`{项目}/产出/数据/{指标名}_分析_{日期}.md` |
| 4 | 手动数据录入什么方式？ | **两种方式都支持**：① 直接在看板添加手动卡片（输入框填数值）；② 通过 CSV 批量上传。用户更倾向 CSV 批量方式 |
| 5 | 分析和执行是否挂钩？ | **不挂钩**。先分析 → 看报告 → 满意后用户手动决定是否执行。中间不自动触发动作 |
| 6 | 飞书多维表格怎么同步？ | **手动触发同步**。配置连接后，用户点击同步按钮拉取最新数据。v2 再加定时自动同步 |
| 7 | 看板放在哪里？ | **顶栏全局导航模块**（`data-module="board"`）。入口全局化方便随时查看；模块内部按项目隔离——顶部「项目选择器」切换，每个项目文件夹绑定一个独立看板，数据不混 |
