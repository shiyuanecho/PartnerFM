# PartnerFM

**本地 AI 智能体工作台** — 集文件管理、多模型配置、多智能体协作、MCP 工具扩展于一体，纯本地运行，浏览器即界面。

## 这是什么

PartnerFM 是一个运行在本地的 AI 工作台。它把一个 IDE 的文件管理、ChatGPT 的对话体验、Claude Code 的 Agent 工具链、以及 MCP 协议的外部连接能力全部放进一个浏览器页面里。

不用装任何浏览器插件，`pip install` 之后打开浏览器就能用。聊天历史、模型配置、文件夹索引全部存在本地，数据完全私有。

## 功能模块

### 📁 文件管理

侧边栏文件树 + 多标签页编辑器，直接读写本地文件系统。

- **多文件夹管理** — 通过 File System Access API 授权任意本地文件夹，侧边栏展开/折叠，拖拽移动文件
- **从访达拖入** — 多选文件夹直接拖入侧边栏一键添加
- **多格式预览** — Markdown 实时渲染+编辑、HTML 内嵌预览（iframe 热更新）、PDF、图片、代码高亮
- **文件操作** — 新建/重命名/删除，右键菜单，剪贴板操作
- **标签页系统** — 多标签页编辑，支持分屏模式，关闭后可恢复
- **IndexedDB 持久化** — 关闭后重新打开自动恢复所有文件夹和标签页
- **快捷搜索** — `⌘P` 快速搜索文件

### 💬 AI 对话

右侧聊天窗口，流式输出，Markdown 渲染+代码高亮。

- **多对话会话** — 多个对话并存，每个会话独立模型和提示词
- **模型切换** — 在对话中随时切换模型（DeepSeek、GPT-4o、Claude 等）
- **文件上下文** — 将文件夹或文件添加到对话上下文，AI 即可直接读取和写入
- **多模态输入** — 支持粘贴/拖拽图片，自动转为 OpenAI Vision 格式
- **流式 Markdown 渲染** — 逐 token 实时渲染，支持 GFM 表格、任务列表、代码高亮
- **对话持久化** — 聊天历史保存在本地 JSON 文件，刷新不丢

### 🤖 Agent 系统

核心能力。Agent 循环 = think → tool_call → tool_result → response，最多 10 轮迭代。

**内置 11 个工具**：

| 工具 | 说明 |
|------|------|
| `list_dir` | 列出目录内容 |
| `read_file` | 读取文件（支持 offset/limit 分段读） |
| `write_file` | 新建或覆盖写入文件 |
| `search_files` | 全文搜索，返回行号+上下文段落 |
| `semantic_search` | 语义向量检索（基于 SQLite + numpy） |
| `file_stats` | 文件统计（数量/大小/扩展名分布） |
| `recent_files` | 按修改时间列最近文件 |
| `web_search` | DuckDuckGo 网页搜索 |
| `web_fetch` | 抓取网页内容 |
| `run_shell` | 执行 shell 命令（含安全沙箱） |
| `invoke_agent` | 调用其他智能体（多 Agent 协作） |

**文件变更红点** — Agent 通过 `write_file` 修改文件后，文件树对应节点自动亮起红点，点击即消。

**可视化工具调用** — Agent 的每步操作以可折叠卡片展示（工具名、参数、结果），执行状态实时更新。

### 🎭 多智能体协作

主 Agent 可通过 `invoke_agent` 工具调度其他注册的智能体。

**预置 5 个智能体**：

| 智能体 | 专长 | 工具白名单 |
|--------|------|-----------|
| 全能助手 | 通用任务，灵活调度 | 全部 10 个工具 |
| 代码助手 | 全栈开发、算法、审查 | 8 个工具 |
| 文案助手 | 技术博客、产品文档、翻译 | 4 个工具 |
| 图表助手 | SVG 图表、架构图、流程图 | 2 个工具 |
| 数据分析师 | 数据清洗、统计、可视化 | 6 个工具 |

每个智能体独立配置：系统提示词 + 模型 + 工具白名单 + 产物输出目录 + 文件类型过滤。用户可在 UI 上创建/编辑 Agent，无需改代码。

**协作模式**：
- **串联**：代码助手写代码 → 文案助手写说明文档 → 图表助手画架构图
- **内嵌**：主 Agent 在任务中调用子 Agent 完成专项工作，结果自动汇总
- 子 Agent 的每步操作同样以可视化卡片展示，可展开查看详情

### 🎯 角色系统

12 个预置角色，分 4 类：

| 分类 | 角色 |
|------|------|
| 内容创作 | 文案写手、教程讲师、翻译润色 |
| 知识整理 | 知识库编辑、阅读助理、数据解读 |
| 自媒体运营 | 选题策划、爆款优化 |
| 通用 | 通用助手、自定义 |

选中角色后，其 prompt 自动注入到 Agent 的 system message。角色数据可通过 UI 增删改。

### 🔧 模型管理

- **内置 Provider 注册表** — 302.AI、DeepSeek、OpenAI、Anthropic、智谱 GLM、通义千问、Moonshot
- **模型元数据展示** — 每个模型显示上下文长度和价格
- **自定义 Provider** — 支持任意 OpenAI 兼容 API
- **API Key 管理** — 密钥仅存本地，支持环境变量兜底
- **302.AI 聚合 API** — 一把 Key 访问 50+ 模型

### 🔌 MCP 集成

支持 MCP（Model Context Protocol）标准协议，用户配置任意 MCP CLI 命令，PartnerFM 自动管理子进程并将工具注册到 Agent。

**预置连接器**：飞书、企业微信、GitHub、Notion、Figma、ima 知识库、LKE 知识引擎

配置后 Agent 可直接调用这些平台的能力（如发飞书消息、读 Notion 文档、查 GitHub Issue），工具命名自动加 `mcp_{server_id}_` 前缀避免冲突。

### 📊 数据监测

通用数据监测模块，支持接入 RedFox 等多数据源平台，查询社媒数据（抖音、小红书等平台的热门内容/话题/创作者），结果以卡片形式渲染。

### 🧠 向量语义检索

- **可配置 embedding 服务** — OpenAI / 302.AI / 任意 OpenAI 兼容厂商
- **SQLite 本地向量索引** — 500 字分块 + 100 字重叠，增量更新
- **语义搜索** — "我去年写的关于定价的笔记" → AI 自动找到对应文件
- **多格式文本提取** — 支持 Markdown、代码、PDF、Word、OCR（可选）
- **无需联网查询** — query 向量在本地做余弦相似度计算，敏感数据不出机器

## 架构

```
┌──────────────────────────────────────────────┐
│              浏览器 (Chrome/Edge)              │
│  ┌──────────┐ ┌──────────┐ ┌───────────────┐ │
│  │ 文件管理  │ │ AI 对话   │ │ 配置管理       │ │
│  │ (单页)   │ │ (流式SSE) │ │ (模型/CLI/MCP) │ │
│  └────┬─────┘ └────┬─────┘ └───────┬───────┘ │
│       │            │               │          │
│  File System   IndexedDB      localStorage    │
│  Access API                               │
└───────┼────────────┼───────────────┼──────────┘
        │            │               │
   HTTP │       SSE  │          REST │
        │            │               │
┌───────┴────────────┴───────────────┴──────────┐
│              Python HTTP Server                │
│  ┌──────────────────────────────────────────┐ │
│  │  /api/agent    SSE Agent 循环（11 工具）  │ │
│  │  /api/chat     SSE 流式对话              │ │
│  │  JSON 文件持久化（配置/历史/日志）        │ │
│  │  SQLite 向量索引（语义搜索）              │ │
│  │  MCP 子进程管理（JSON-RPC over stdio）    │ │
│  └──────────────────────────────────────────┘ │
└──────────────────────────────────────────────┘
```

- **前端**：单文件 HTML（~11,000 行），零构建零依赖，marked.js + highlight.js CDN 引入
- **后端**：Python 标准库 HTTP Server（~3,600 行），无框架依赖
- **数据**：JSON 文件 + SQLite + IndexedDB + localStorage，全部本地

## 安装

```bash
pip install partnerfm
```

可选依赖（PDF/Word 文本提取、语义检索、OCR）：

```bash
pip install partnerfm[full]
```

### 依赖说明

| 依赖 | 作用 | 必需 |
|------|------|------|
| Python 3.9+ | 运行服务端 | ✓ |
| numpy | 向量余弦相似度计算 | 语义检索需要 |
| pdfplumber | PDF 文本提取 | 索引 PDF 需要 |
| python-docx | Word 文本提取 | 索引 docx 需要 |
| pytesseract + tesseract CLI | 图片 OCR | 索引图片需要 |

## 启动

```bash
partnerfm --open
# → 浏览器自动打开 http://localhost:8765
```

或源码运行：

```bash
python3 server.py
```

Mac 用户也可双击 `启动.command` 一键启动。

## 使用流程

1. **添加文件夹** — 点击左下角 `+` 按钮，选择一个本地文件夹作为工作区
2. **配置模型** — 切换到「模型」模块，填入 API Key，勾选要使用的模型
3. **开始对话** — 在右侧聊天窗口输入问题，Agent 会读取工作区文件并回答
4. **协作模式** — 在对话中说"让代码助手写一个排序算法，然后让文案助手写说明文档"，主 Agent 会自动调度子 Agent 完成
5. **查看红点** — Agent 修改文件后，文件树对应节点出现红色圆点，点击即消

## 浏览器兼容

需要 Chrome / Edge 108+（File System Access API）。Safari / Firefox 暂不支持。

## 数据隐私

- 所有数据（聊天记录、模型配置、索引数据库）**仅存储在本地**
- LLM API 调用直接从前端发到用户配置的 API 端点，不经过任何中间服务器
- 语义检索的 query 向量在本地 SQLite 做相似度计算，**不发送到云端**
- API Key 支持环境变量兜底，不会明文存入配置文件
