# 看板改版方案：数据源主导模式

## 1. 问题陈述

### 1.1 现状

当前看板以「项目」为主导。用户进入看板后，第一件事是选择一个 workflow project，然后在该项目的上下文里添加数据源、配置指标卡片。

数据源「嵌套」在项目配置文件中：

```
dashboards/
├── .dashboard-sz-app.json     ← datasources 放在这里
└── .dashboard-learn-cn.json   ← datasources 又放一遍（如果重复）
```

### 1.2 问题

1. **入口逻辑反了**：用户最关心的是「我的增长数据怎么样了」，不是「某个项目怎么样了」。项目只是数据的执行载体。
2. **共享数据要复制**：一个飞书报表同时反映到两个项目的数据上时，配置要复制两份，不同步。
3. **切换场景成本高**：独立开发者同时做 2-3 个项目，每天切换项目看数据很累赘。
4. **聊天上下文断链**：聊天时选文件夹作为上下文，但文件夹和数据源是两个系统，无法联动。

### 1.3 不改的成本

- 后续加新项目时，每个项目都要重新配置数据源绑定
- 数据分析 Agent 无法自动获得项目对应的业务数据作为参考
- 产品逻辑和用户真实工作流之间的错位持续存在

---

## 2. 设计原则

1. **数据源是一级实体**：数据源独立存在，不隶属于任何项目/文件夹
2. **文件夹是标签**：文件夹只是数据源的一个维度标签，不决定数据源的生命周期
3. **数据为中心**：看板的默认视图是所有数据源的总览，而不是某个文件夹的视图
4. **文件夹筛选是下钻**：需要在具体文件夹上下文里看数据时，通过筛选器下钻
5. **凭证分离**：飞书 App ID / Secret 在配置模块统一管理，多个数据源共享同一凭证
6. **URL 直达**：添加数据源只需粘贴飞书多维表格链接，系统自动解析提取 appToken、tableId，并自动用表格名称作为数据源名

## 3. 数据模型

### 3.1 飞书凭证配置：`.seegent-feishu-credentials.json`

**位置**：`Seegent/.seegent-feishu-credentials.json`
**作用**：飞书 App ID / Secret 只存一次，多个数据源共享。在配置模块的「飞书凭证」子标签管理。

```json5
{
  "credentials": {
    "<credentialId>": {
      "name": "字节飞书",              // 凭证名称（用户自定义）
      "domain": "bytedance.feishu.cn", // 域名，用于 URL 自动匹配
      "appId": "cli_a2b3c4d5",
      "appSecret": "<placeholder>"
    }
  }
}
```

域名匹配规则：当用户粘贴链接 `https://bytedance.feishu.cn/base/...` 时，自动从域名匹配到 `"字节飞书"` 凭证。

### 3.2 数据源注册表：`.seegent-datasources.json`

**位置**：`Seegent/.seegent-datasources.json`
**作用**：所有数据源的唯一存储位置。**唯一支持的类型：飞书多维表格。**

```json5
{
  "version": 3,
  "datasources": {
    "<sourceId>": {
      // === 基本元信息 ===
      "name": "小红书运营数据",          // 自动取自多维表格名称
      "type": "feishu",
      "createdAt": "2026-06-29T04:30:00Z",
      "updatedAt": "2026-06-29T12:00:00Z",

      // === 绑定关系 ===
      "boundFolders": ["<folderId>", ...],  // 可绑 0~N 个文件夹。添加数据源时不绑定，后续单独操作

      // === 连接参数 ===
      "credentialId": "字节飞书",          // 引用飞书凭证 ID（App ID/Secret 在凭证配置中）
      "appToken": "BASCxxxxx",             // 从 URL 自动提取（/base/ 后）
      "tableId": "tblYYYYY",              // 从 URL 自动提取（?table=）
      "sourceUrl": "https://...",          // 原始链接（备份）

      // === 列映射 ===
      "columnMapping": {
        "dateField": "日期",
        "metrics": [
          { "name": "粉丝数", "field": "粉丝数", "unit": "" },
          { "name": "曝光量", "field": "曝光量", "unit": "" }
        ]
      },

      // === 缓存数据 ===
      "rawData": [
        { "date": "2026-06-01", "粉丝数": 12000, "曝光量": 45000 }
      ],
      "lastSyncAt": "2026-06-29T12:00:00Z"
    }
  }
}
```

### 3.3 看板配置文件：`dashboards/.dashboard-<folderId>.json`

**位置**：`Seegent/dashboards/.dashboard-<folderId>.json`
**作用**：每个文件夹的看板配置，只包含卡片布局和分析记录。数据源从注册表引用。

```json5
{
  "version": 2,
  "folderId": "<folderId>",

  // 卡片配置
  "cards": [
    {
      "id": "card-001",
      "sourceId": "feishu-stats",         // 引用注册表中的数据源 ID
      "metricKey": "粉丝数",
      "type": "number_trend",             // 统计数字 + 趋势图
      "displayOptions": {
        "title": "粉丝增长",
        "format": "number",               // "number" | "percent" | "currency"
        "period": "7d"                    // 显示近 7 天
      }
    }
  ],

  // Agent 分析记录
  "analyses": [
    {
      "id": "analysis-001",
      "sourceId": "feishu-stats",
      "generatedAt": "2026-06-29T10:00:00Z",
      "summary": "近 7 天粉丝增长 8%，曝光增长 12%...",
      "fullReportPath": "/产出/数据/分析报告-20260629.md"
    }
  ]
}
```

### 3.3 数据关系总览

```
.seegent-datasources.json            datasources 注册表（唯一存储）
├── sourceId: "feishu-stats"
│   ├── boundFolders: ["A", "C"]       ← 多文件夹绑定
│   ├── config: {飞书连接配置}
│   └── rawData: [时序数据]
│
└── sourceId: "feishu-lc"
    ├── boundFolders: ["B"]
    ├── config: {...}
    └── rawData: [...]

dashboards/
├── .dashboard-A.json                 文件夹「识字App」的看板
│   └── cards: [{sourceId: "feishu-stats", ...}]   ← 引用注册表
│
├── .dashboard-B.json                 文件夹「Learn Chinese」的看板
│   └── cards: [{sourceId: "feishu-lc", ...}]
│
└── .dashboard-C.json                 文件夹「小红书运营」的看板
    └── cards: [{sourceId: "feishu-stats", ...}]   ← 和 A 引用同一个数据源
```

### 3.4 folderId 的生成和持久化

- **folderId** = IndexedDB 中 `saveFolderHandle()` 存储的 handle ID（`FileSystemDirectoryHandle` 的 `name` 属性 + UUID 后缀）
- handle ID 不依赖文件路径，文件夹改名/移动不失效
- 在 `addFolder()` 时生成并持久化到 IndexedDB

#### 4.1.1 筛选器行为

- 默认：`全部文件夹` → 显示所有数据源（无论绑定到哪个文件夹）
- 选中 `识字App` → 只显示 `boundFolders` 包含该文件夹的数据源
- 选中 `小红书运营` → 只显示绑定了小红书运营的数据源
- 如果文件夹没有绑定任何数据源，在筛选器中仍显示，但选中后显示空状态

### 4.2 添加数据源流程

**唯一支持的数据源类型：飞书多维表格。** 用户只需粘贴链接，系统自动解析提取、匹配凭证、用表格名作为数据源名。

```
[+ 添加数据源]
    ↓
粘贴飞书多维表格链接
  https://bytedance.feishu.cn/base/BASCxxxxx?table=tblYYYYY
    ↓
系统自动解析：
  appToken = BASCxxxxx         ← 从 /base/ 后提取
  tableId  = tblYYYYY          ← 从 ?table= 提取
  domain   = bytedance.feishu.cn → 匹配「字节飞书」凭证
    ↓
如果没有匹配的凭证 → 提示去配置模块添加
    ↓
[测试连接]
  → 获取 tenant_access_token（用凭证中的 App ID / Secret）
  → 拉取表格字段列表
  → 自动读取表格名称作为数据源名（例如"小红书运营数据"）
    ↓
列映射（和现在一样）
  选择日期列 + 选择要追踪的指标列
    ↓
确认保存
  → 写入 .seegent-datasources.json
  → 数据源以 boundFolders: [] 形式存在
  → **不绑定文件夹**，之后单独操作
```

#### 4.2.1 URL 解析规则

飞书多维表格链接格式：

| 链接示例 | 提取的 appToken | 提取的 tableId |
|---------|----------------|----------------|
| `feishu.cn/base/BASCxxxxx?table=tblYYYYY` | `BASCxxxxx` | `tblYYYYY` |
| `feishu.cn/base/BASCxxxxx?table=tblYYYYY&view=vewZZZZZ` | `BASCxxxxx` | `tblYYYYY` |

域名匹配到凭证配置中的 `domain` 字段。如果匹配失败，弹出提示"未找到匹配的飞书凭证，请先在配置模块添加"。

#### 4.2.2 数据源名称自动获取

在测试连接阶段，调用飞书 API 获取表格元数据（`GET /bitable/v1/apps/{app_token}/tables/{table_id}`），从返回中读取 `name` 字段作为数据源名称。用户也可手动修改。

### 4.3 绑定文件夹（独立操作）

添加数据源时 **不绑定文件夹**。绑定是数据源卡片上的一个独立操作。

每个数据源卡片底部显示：
```
📁 绑定: 识字App, 小红书运营  [编辑绑定] [删除]
```

点击 [编辑绑定] → 弹出多选列表 → 勾选/取消勾选 → 保存 → 更新 `datasources[sourceId].boundFolders`

### 4.4 无绑定数据源的文件夹

如果文件夹没有任何数据源绑定：
- 看板筛选器中可选该文件夹
- 选中后显示空状态：

```
┌──────────────────────────────┐
│                              │
│  📂  识字App                 │
│                              │
│  此文件夹暂未绑定数据源       │
│                              │
│  [+ 添加数据源并绑定到此文件夹] │
│                              │
└──────────────────────────────┘
```

### 4.5 数据源的删除

删除数据源时：
1. 从 `.seegent-datasources.json` 中移除条目
2. 遍历所有看板配置，删除引用该 `sourceId` 的卡片和分析记录
3. 如果某个文件夹因此变成无数据源状态，不删看板配置，保留空壳

## 6. 聊天联动逻辑

当用户在聊天模块选择文件夹后，上下文注入规则：

```
1. 用户选择文件夹「识字App」(folderId: "A")
2. 系统查询 .seegent-datasources.json
   → 找到 boundFolders 包含 "A" 的所有数据源
3. 对每个匹配的数据源，提取最近 7 天各指标的 summary
4. 构造成如下上下文注入到 system message：

──── 业务数据参考：识字App ────
数据源：小红书运营数据
├─ 粉丝数：12,300（近 7 天 ↑ 8%）
├─ 曝光量：45,200（近 7 天 ↑ 12%）
├─ 互动率：3.8%（近 7 天 ↓ 1.2%）
└─ 收藏数：1,234（近 7 天 ↑ 5%）
──── 文件参考（文件夹内）────
├─ src/  (12 个文件)
├─ docs/ (3 个文件)
└─ ...最新修改：index.html（30 分钟前）
```

这样 Agent 同时知道 **业务数据** + **项目文件**，产出内容时自然参考实际数据。

## 7. 向后兼容与数据迁移

旧看板目前还没有实际数据，此轮改版从零开始设计，无需迁移。

只需确保：新代码上线后，旧的 `dashboards/` 目录下残留的空文件不会被误读即可（加 version 字段校验，跳过 version < 2 的文件）。

### Phase 1：数据层 + 凭证模块

| 项 | 说明 |
|----|------|

| 新增 per-folder 看板配置格式 | `.dashboard-{folderId}.json` |

| 项 | 说明 |

## 9. 用户故事（按 Phase 排序）

### Phase 1（P0）

- 作为独立开发者，我在配置模块添加飞书凭证（App ID / Secret + 域名），一次配置全局复用
- 作为独立开发者，我粘贴飞书多维表格链接，系统自动解析提取 appToken、tableId，并匹配到对应的飞书凭证
- 作为独立开发者，添加数据源时系统自动用表格名称作为数据源名
- 作为独立开发者，进来看板时默认看到所有数据源的总览，不显示项目选择器
- 作为独立开发者，筛选文件夹后只显示该文件夹绑定的数据源和指标卡片

### Phase 2（P1）

- 作为独立开发者，我可以在数据源卡片上编辑绑定到哪些文件夹，灵活调整覆盖范围
- 作为独立开发者，我在没有数据源的文件夹下看到空状态引导，知道该做什么
- 作为独立开发者，我可以在数据源卡片上直接「重新同步」一键拉取最新数据
- 作为独立开发者，删除数据源时系统自动清理所有文件夹中引用的卡片

### Phase 3（P2+）

- 作为独立开发者，我在聊天中选择文件夹时，自动获得该文件夹绑定的数据源作为 Agent 上下文
- 作为独立开发者，我可以在数据源上点击 [⚡分析]，Agent 自动基于当前数据生成分析报告
