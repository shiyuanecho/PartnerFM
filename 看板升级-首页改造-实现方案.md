# 📊 看板升级为每日工作首页 — 实现方案

## 背景

独立开发者用 Seegent 管理多个项目（识字 App、Learn Chinese 等）。工作流：

> 早上打开 Seegent → 看各项目数据 → 看待办 → 进入项目 → 开始工作（文件 + AI 对话）→ 看产出物

目前 Seegent 默认打开"文件管理"模块，看板只是侧边栏的一个 tab。**希望打开 Seegent 时第一眼看到的是看板**，作为每天工作的入口。

## 需求范围

### 一、必须做（P0）

#### 1. 看板设为首页

- `activeModule` 默认值从 `'file'` 改为 `'board'`
- 首次打开 Seegent 直接看到看板，不是文件管理
- 已在 localStorage 存了上次模块的老用户不受影响

#### 2. 项目卡片增强

现在的卡片显示：项目名、文案、文件夹路径。

增强后每个卡片应显示：

- 项目名称 + 图标（emoji）
- **核心指标**（来自数据源）：如"关注 333 ↑12"、"点赞 11 ↑6"——只显示有数据的指标
- **待办数徽章**：如"3 个待办"
- **最后活动时间**：如"上次活动：2 小时前"
- 底部操作区：「进入项目」「查看文件」

无数据源的项目卡片也要正常显示（指标处显示"暂无数据，连接数据源"）。

点击"进入项目"→ 自动切换到该项目的工作区（切换到文件模块 + 展开对应文件夹）。

#### 3. 待办区

- 在看板页面中加入"待办"区块，显示所有项目的待办汇总
- 数据来源：后端 `/api/board/projects` 已经返回每个项目的 `todos` 数组
- 显示格式：勾选框 + 待办文字 + 所属项目标签 + 日期
- 支持勾选完成（前端状态切换，暂不持久化）
- 空状态：显示"暂无待办"，引导文案"在项目中创建 TODO.md 文件即可自动识别"

#### 4. 最近活动区

- 显示最近 5 条文件活动（修改/创建的文件）
- 显示格式：时间 + 图标（文件类型）+ 文件名 + 所属项目标签
- 数据来源：`/api/board/projects` 返回的 `fileTree` 找 mtime 最新的
- 空状态：显示"还没有活动记录，开始第一个工作吧"

#### 5. 快速入口

在看板顶部显示 2-3 个快捷按钮：

- 「💬 继续上次对话」→ 切换到聊天模块 + 打开最近一次会话
- 「📄 最近产出物」→ 切换到文件模块 + 展开产出目录

### 二、不要做（Non-Goals）

- ❌ 不新建后端 API（复用现有 `/api/board/projects` 和 `/api/dashboard/*`）
- ❌ 不改动设置模块（模型/角色/MCP 保持原样）
- ❌ 不实现新数据源类型（只支持已有的飞书多维表格）
- ❌ 不实现待办持久化（勾选状态暂不存盘）
- ❌ 不改动文件模块、聊天模块
- ❌ 不做项目工作流追踪

### 三、可选增强（P1，时间充裕再做）

- 项目卡片按"最后活动时间"排序
- 待办数在侧边栏「看板」按钮上也显示 badge
- 看板页面顶部显示"早上好/下午好"问候语
- 最近活动支持展开查看详情

### API 返回数据结构

`GET /api/board/projects` 返回：

```json
{
  "projects": [
    {
      "folderId": "...",
      "folderName": "识字App",
      "workspacePath": "/Users/.../识字App",
      "todos": [
        { "line": "完成首页改版", "source": "TODO.md", "checked": false }
      ],
      "fileTree": { ... },
      "folderCount": 12,
      "fileCount": 45
    }
  ]
}
```

`todos` 数组已返回（可渲染待办区）。`fileTree` 可遍历找 mtime 最新的文件（渲染最近活动区）。

### 约束

- 保持现有 CSS 变量体系（`--bg-primary`, `--text-primary` 等）
- 卡片样式参考现有 `.dash-project-card` 类
- 每个区域都要有空数据时的友好提示
- 模块数量不新增（文件/聊天/看板/模型/角色 5 个不变）
